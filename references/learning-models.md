# Learning Models

Use learning models as mechanisms inside tasks, tests, and project work. Do not name a model unless it changes what the learner does.

## Feynman Explanation

Mechanism:

1. Explain the concept in plain language.
2. Identify unclear terms.
3. Use an example.
4. Rewrite the explanation.

Use for: daily output, glossary entries, self-test corrections, final showcase.

Task pattern:

```md
Explain today's concept to a smart beginner in 150 words. Then list what still feels vague.
```

## Socratic Questioning

Mechanism:

- define terms
- surface assumptions
- ask for evidence
- test counterexamples
- compare alternatives
- find consequences

Use for: expert room, stage tests, project critique, replan.

Task pattern:

```md
What must be true for your answer to work? What would prove it wrong?
```

## Deliberate Practice

Mechanism:

1. Isolate one weak subskill.
2. Practice it with a clear standard.
3. Get feedback.
4. Repeat with a harder variant.

Use for: exercises, mistake book, project iteration.

## Retrieval Practice And Spaced Review

Mechanism:

- test before rereading
- record wrong answers
- schedule review
- retest weak concepts

Use for: daily quizzes, stage tests, glossary review.

Review cadence: same day, +2 days, +7 days, next stage test.

## Bloom Mastery Ladder

Move concepts through:

1. Remember: define.
2. Understand: explain.
3. Apply: solve a case.
4. Analyze: compare and diagnose.
5. Evaluate: critique a solution.
6. Create: use it in the final project.

Every core concept should reach at least Apply. Project-critical concepts should reach Create.

## Kolb Learning Cycle

Mechanism:

1. Concrete experience.
2. Reflective observation.
3. Abstract model.
4. Active experiment.

Use for: project logs and weekly retrospectives.

## Project-Based Learning

Mechanism:

- choose a small final project
- start on Day 1
- turn daily outputs into project parts
- ship a visible version on Day 30

The project is not extra homework. It is the container that proves learning happened.

## Metacognitive Review

Daily review fields:

- What did I understand better today?
- What did I still confuse?
- What evidence shows progress?
- What should change tomorrow?

Use this to update `progress.md`, `mistake_book.md`, and future task difficulty.

## Scaffolding And Zone Of Proximal Development

Keep tasks slightly above the learner's current ability. If the learner fails twice, reduce difficulty and add a worked example. If the learner succeeds easily twice, increase complexity or connect the concept to the project.

## Model Routing

| Need | Use |
|---|---|
| Concept feels vague | Feynman + Socratic |
| User keeps making errors | Deliberate practice + mistake book |
| User forgets | Retrieval + spaced review |
| User can explain but not use | Bloom Apply/Analyze |
| User needs real output | Project-based learning |
| User feels stuck | Scaffolding |
| Plan no longer fits | Metacognitive review + replan |
