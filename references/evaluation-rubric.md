# Evaluation Rubric

Use this to validate a generated learning repo or score progress.

## Repository Score

```md
结构完整性：0-20
目标适配：0-15
学习模型落地：0-15
每日可执行性：0-15
测试与反馈闭环：0-15
项目牵引：0-10
事实边界：0-5
长期续航：0-5
总分：100
```

## Pass Criteria

Publishable repo:

- score >= 85
- all P0 files exist
- Day 1 is complete
- Day 7/14/21/30 tests exist
- every daily task has an output
- final project starts on Day 1
- high-risk/current facts are marked

Do not call the repo complete if:

- it is only a plan
- `AGENTS.md` or `progress.md` is missing
- Day 1 has no self-test
- there is no final project
- learning models appear only as names
- expert room has no action or repository update

## Daily Learning Score

```md
理解：0-20
练习：0-20
输出物：0-20
测验：0-20
复盘：0-10
项目推进：0-10
总分：100
```

Below 60: repeat or simplify the day.  
60-79: continue, but add one corrective exercise.  
80-89: continue normally.  
90+: increase challenge or project depth.

## Stage Test Score

Stage tests must include:

- concept understanding
- application
- case analysis
- project advancement
- reflection

Write weak areas to `06_mistakes/mistake_book.md` and next adjustments to `progress.md`.
